python-3-7-notes.txt

Ubuntu 18.04 users are lucky that 3.7 is in the repos. If you are on a different
distro check to see if your repo just installs an older version but has newer.

On Ubuntu 18.04 you would do this

# you will get some error messages about not being able to lock /var/cache
# and /var/lib
#
sudo apt-get install --no-install-recommends -y software-properties-common \
            python3.7 python3.7-venv python3.7-distutils &&
            apt-get clean && rm -rf /var/lib/apt/lists/*

sudo update-alternatives --install /usr/bin/python3 python3 /usr/bin/python3.6 1
sudo update-alternatives --install /usr/bin/python3 python3 /usr/bin/python3.7 2

sudo update-alternatives --config python3
There are 2 choices for the alternative python3 (providing /usr/bin/python3).

  Selection    Path                Priority   Status
------------------------------------------------------------
  0            /usr/bin/python3.6   3         auto mode
  1            /usr/bin/python3.6   3         manual mode
* 2            /usr/bin/python3.7   2         manual mode

choose the highest number

python3 -V

should now show 3.7 or whatever you installed

other distros will have other versions, possibly in different spots.

# upgrade pip to latest - note you will need to know version from above.
# Ignore warning about .local/bin not being in path. When you reboot
# your .profile will take care of that.
#
sudo curl -s  https://bootstrap.pypa.io/pip/3.7/get-pip.py -o get-pip.py && \
    python3 get-pip.py --force-reinstall && \
    rm get-pip.py

# reboot before installing this.
#
python3 -m pip install MarkupSafe
